# Content Expansion Skill

## 触发示例

```
/content-expand 车型CR-V，50篇今日头条，方向春节返乡+家庭用车
```

---

## 工作流程

### Step 1: 材料消化

1. 解析 Brief：车型、平台、数量、方向
2. 校验种子库 `01-输入材料/既定资料/{车型}.md`，无则停止并提示上传
3. 输出《材料理解摘要》

### Step 2: 需求确认（门禁）

向用户提问确认：卖点优先级、目标人群、差异化判断。3题全答才继续。

详见 → `references/tone-guide.md`

### Step 3: 多样化分配

预分配 **风格×角度×话题×开头类型×论证结构×核心数据点×收尾方式** 7维组合，输出完整分配表。

分配表必须在本步骤一次性生成完毕，后续创作严格按表执行。

详见 → `references/diversity-matrix.md`

#### 并行代理模式（≥10篇时自动启用）

当任务总量 ≥ 10篇时：

1. 在 Step 3 完成分配表后，将任务拆分为 N 个子批次（每批次 3-5 篇）
2. 每个子批次分配到独立的子代理执行，子代理只接收：
   - 本批次的分配表行（含7维锁定）
   - 对应风格的语料样本（只发1-2个，不发全库）
   - 种子库中与本批次核心数据点相关的段落
   - 质量标准和铁律
3. 子代理之间无共享 context，物理隔离防止自我复制
4. 所有子代理完成后，主流程执行跨批次查重，发现重复则标记返工

### Step 4: 批量创作

**节奏**：抖音5篇/批次，今日头条3篇/批次，单次对话上限15篇。

**参考资源**（按需读取）：

| 资源     | 路径                                   | 说明         |
| -------- | -------------------------------------- | ------------ |
| 种子库   | `01-输入材料/既定资料/`                | 所有数据来源 |
| 抖音语料 | `02-参考学习/风格语料库/抖音专属/`     | 抖音必读     |
| 头条语料 | `02-参考学习/风格语料库/今日头条专属/` | 今日头条必读 |
| 车型语料 | `02-参考学习/风格语料库/车型语料.md`   | 本田车型卖点 |
| 开头类型 | `02-参考学习/风格语料库/开头变体库.md` | 开头类型定义 |
| 热梗神评 | `02-参考学习/热梗库/` `神评库/`        | 每5篇融入1个 |
| 平台规范 | `references/platform-rules.md`         | 字数、格式   |
| 质量标准 | `references/quality.md`                | 审校清单     |

**创作铁律（每篇生成时强制遵守）**：

> STOP — 写任何一段前检查：
>
> - 同一段落内是否用"XX这块/方面"描述了2个以上卖点？→ 拆到不同段落
> - 是否出现"首先...其次...最后"？→ 去掉序号词，用场景/转折自然过渡
> - 是否连续3句以上用相同句式？→ 打乱结构
> - 是否出现禁用AI句式（但问题来了/算笔账/说白了/市场已经给出了答案）？→ 重写
> - 是否以车主身份叙述？→ 切换为旁观者/行业观察者视角

详见 → `references/quality.md` 禁用词+人话化标准

**产出格式**：

- 抖音：250-350字，三段式标题，发布文案+标签
- 今日头条：650-800字，单行标题

### Step 4.5: 后置事实自检（每篇生成后立即执行）

逐篇检查以下4项，不通过则当场重写：

1. **数据口径一致性**：标题提到的数据范围 = 正文使用的数据范围（如标题说"三年"，正文不能出现"五年"）
2. **场景逻辑一致性**：开头设定的场景 = 全文贯穿的场景（如开头说"高速服务区"，正文不能出现"4S店蹲了一天"）
3. **词汇准确性**：搜索"购房/住房/房贷"等非汽车词汇 → 0个
4. **跨篇查重**：当前篇的开头前20字 ≠ 本批次任何已完成篇的开头前20字

### Step 5: 批量审校

每篇过审校清单，重点检查：搜索"方面/这块"同段≤1个，"首先/其次/第三"=0个。

详见 → `references/quality.md`

### Step 5.5: 确认与输出

用户确认后，生成 Excel。详见 → `references/platform-rules.md`

### Step 6: 上下文管理

每15篇强制交接，输出《任务交接文档》。详见 → `references/handover-protocol.md`

---

## 六大铁律

1. **不编造** — 信息来自既定资料
2. **不第一人称** — 第三客观视角
3. **不结构化** — 禁用"XX方面/首先/其次"
4. **不广告腔** — 禁用极致/震撼/硬核
5. **不同质** — 每篇组合不同
6. **不车主** — 禁止以车主身份写作，始终保持旁观者/行业观察者视角
