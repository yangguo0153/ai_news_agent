# 客户经理 - 需求消化模板

> 用途：帮助客户经理快速消化品牌方需求，转换为内部 Brief
> 使用场景：接收用户输入后的第一步

---

## 一、需求信息采集

### 1.1 基础信息（必填）

```markdown
【车型】：_________（如：CR-V）
【平台】：_________（如：抖音/今日头条）
【数量】：_________（如：15篇）
【方向】：_________（如：春节返乡/家庭用车）
```

### 1.2 补充信息（选填）

通过与用户对话，补充以下信息：

**传播侧重点**：
- [ ] 情感共鸣（温情路线）
- [ ] 理性对比（数据路线）
- [ ] 场景展示（生活路线）

**目标用户画像**：
- [ ] 宝妈（带娃出行）
- [ ] 上班族（通勤代步）
- [ ] 孝子（接送父母）
- [ ] 小夫妻（第一辆车）
- [ ] 事业有成（品质生活）

**核心卖点偏好**：
- [ ] 空间（MM理念、后备箱）
- [ ] 安全（10气囊、ACE车身）
- [ ] 智能（Honda SENSING 360）
- [ ] 省心（品质、保值）
- [ ] 舒适（内饰、人机工程）

## 二、需求确认话术

### 2.1 开放式问题（引导用户表达）

```
1. "您希望这批内容重点突出什么？"
   → 引导用户说出核心诉求

2. "目标用户是谁？他们的痛点是什么？"
   → 引导用户描述用户画像

3. "有没有特别想强调的产品特点？"
   → 引导用户说出卖点偏好

4. "内容调性上，您希望是温情的、理性的，还是幽默的？"
   → 引导用户明确调性
```

### 2.2 选择题（快速确认）

```
1. "平台是抖音还是今日头条？"
   → 确认平台（影响字数和风格）

2. "数量是5篇、10篇还是15篇？"
   → 确认数量（影响多样性策略）

3. "方向是春节返乡、家庭用车，还是其他？"
   → 确认方向（影响场景选择）
```

```json
{
  "车型": "CR-V",
  "平台": "抖音",
  "数量": 15,
  "方向": "春节返乡",
  "侧重点": "情感共鸣",
  "目标用户": ["宝妈", "孝子", "小夫妻"],
  "核心卖点": ["空间", "安全"],
  "调性": "温和喜庆",
  "特殊要求": "避免直白描述，多用场景暗示"
}
```

## 四、常见需求类型

### 类型1：节日营销（春节/国庆/中秋）

**特征**：
- 时间敏感，需要节日氛围
- 情感共鸣为主，理性为辅
- 场景：返乡、团聚、出游

**建议调性**：温和喜庆、场景化

---

### 类型2：产品上市/改款

**特征**：
- 突出新功能、新配置
- 理性对比为主，情感为辅
- 场景：试驾、对比、升级

**建议调性**：专业理性、数据支撑

---

### 类型3：日常种草

**特征**：
- 长期传播，不追热点
- 生活场景为主
- 场景：通勤、接娃、周末

**建议调性**：生活化、真实感

---

## 五、注意事项

### 5.1 禁止事项

- ❌ 不要直接接受模糊需求（"随便写点"）
- ❌ 不要自行脑补用户意图
- ❌ 不要跳过确认环节

### 5.2 必做事项

- ✅ 必须与用户确认 Brief
- ✅ 必须明确调性和侧重点
- ✅ 必须输出结构化 JSON
