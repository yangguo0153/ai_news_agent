## 二、重构原则

### 2.1 分层披露

**原则**：不同角色只看到需要的材料

| 角色 | 可见材料 | 不可见材料 |
|------|---------|-----------|
| 客户经理 | 需求消化模板 | 其他所有材料 |
| 策划者 | 传播方向案例库 | Writer/审核者材料 |
| Writer | 结构模板+内容变量库 | 完整范文 |
| 审核者 | 硬性规则清单 | Writer材料 |
| 输出校订者 | Anti-AI-style特征库 | 其他所有材料 |

**优势**：
- 避免信息过载
- 减少照抄风险
- 提高执行效率

---

### 2.2 从"范文"到"规则"

**原则**：不给完整范文，给提炼后的规则

**对比**：
```
❌ 旧方式：
给 Writer 看完整范文（500字）
→ AI 容易照抄

✅ 新方式：
给 Writer 看：
- 结构模板（只有框架）
- 细节描写库（随机抽取3-5个）
- 口吻样本库（随机抽取2个）
→ AI 学习"感觉"，不照抄
```

---

### 2.3 从"正例"到"反例"

**原则**：告诉 AI "什么是不好的"，而非"什么是好的"

**对比**：
```
❌ 旧方式：
告诉 AI："好的内容是温和喜庆的"
→ AI 不知道怎么做

✅ 新方式：
告诉 AI："禁止使用'智商税''买教训'等激进表达"
→ AI 知道什么不能做
```
