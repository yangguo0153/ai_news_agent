# CRV抖音脚本修复计划

## 问题诊断

### 用户反馈的核心问题
1. **内容逻辑混乱** - 前言不搭后语，段落之间缺乏自然过渡
2. **参数堆砌严重** - 单篇内容塞入过多参数，缺乏聚焦
3. **正确标准** - 每篇应围绕1-2个核心参数展开，深入"唠唠"

### 根本原因分析

#### 问题1：Writer Agent 参数分配错误
- **现状**：每篇分配了 `data_point` 但实际上 Writer 把多个参数都塞进去了
- **示例**：一篇里同时出现 "10气囊+590Mpa钢+HC4.0+销量19万" 等4-5个参数
- **后果**：信息过载，没有深度，变成参数罗列

#### 问题2：缺乏"单一卖点深入"约束
- **现状**：Writer Prompt 只要求"首句点题"，没要求"全文聚焦"
- **缺失**：没有明确要求"整篇只围绕1-2个参数展开论述"
- **后果**：Writer 为了"丰富内容"而堆砌各种卖点

#### 问题3：过渡句使用生硬
- **现状**：强制插入过渡句库中的句子，但前后内容不衔接
- **示例**："不过现实情况有点复杂" 后面接的并不是复杂情况的解释
- **后果**：逻辑断裂，前言不搭后语

#### 问题4：语料库参考方式错误
- **用户怀疑**：直接打乱复用语料库内容
- **实际情况**：Writer 确实过度依赖语料库的句式结构，导致内容拼凑感强
- **后果**：内容像是"拼接"而非"原创思考"

## 修复策略

### 策略1：严格单篇参数限制
**新约束**：每篇只能使用 **1个核心数据点** + **最多1个辅助数据点**

| 原分配 | 修复后 |
|--------|--------|
| data_point: 10安全气囊（但文中出现5个参数） | 只讲10安全气囊，其他参数一律不提 |
| data_point: 保值率60.27%（但文中出现销量+配置） | 只讲保值率，用销量作为辅助论证 |

### 策略2：重写 Writer Prompt
**新增强制要求**：
```
【核心约束 - 违反则不合格】
1. 全文只能围绕【1个核心数据点】展开
2. 最多使用【1个辅助数据点】作为支撑
3. 其他所有产品数据（即使你知道）也不许出现
4. 目标：把一个参数讲透，而不是讲一堆参数

【逻辑连贯要求】
1. 每段之间必须有明确的逻辑关系（因果/递进/对比）
2. 禁止生硬插入无关过渡句
3. 段落顺序必须符合认知逻辑：问题→解答→证据→总结
```

### 策略3：重新设计内容结构
**每篇标准结构**：
```
【开头】点题（核心数据点一句话）
【第一段】场景引入 - 这个数据解决什么问题
【第二段】深入解释 - 这个数据是什么意思，为什么重要
【第三段】证据支撑 - 用辅助数据/场景/案例证明
【结尾】收束回归 - 回到开头的数据，给出结论
```

### 策略4：废除过渡句库硬插入
**改为**：让 Writer 根据内容逻辑自然过渡，而非强制插入预制句式

## 关键改进点

### 改进1：从"多参数罗列"到"单参数深度"
**错误示例**（原）：
```
CR-V全系标配10个安全气囊...ACE承载式车身...Honda SENSING 360+...保值率60.27%...
```

**正确示例**（新）：
```
标题：CR-V的10个安全气囊，春节带家人出行够不够用？

脚本：
春节开车带家人出行，安全气囊是最后一道防线。

CR-V全系标配10个安全气囊。这是什么概念？
前排正面2个，前排侧面2个，前排膝部2个，
后排侧面2个，前后贯穿式气帘2个。
从膝盖到头顶，全方位覆盖。

膝部气囊很多同级车没有。别小看这两个气囊，
正面碰撞时，膝盖最容易撞伤。有了它，前排两个人都多一层保护。

春节高速上，车多路况复杂。万一遇到紧急情况，
这10个气囊就是家人安全的最后保障。

10个气囊不是摆设，是实实在在的守护。
```

### 改进2：从"生硬过渡"到"自然逻辑"
**错误示例**（原）：
```
不过现实情况有点复杂。买车容易养车难。
```
（"复杂"在哪里？没有解释，逻辑断裂）

**正确示例**（新）：
```
有人觉得10个气囊用不上。但开车上路，万一遇到紧急情况，气囊就是最后的防线。
```
（明确回应前面的"用不上"观点，逻辑连贯）

### 改进3：从"打乱复用语料"到"原创深度思考"
**要求**：
- 禁止使用语料库中的现成句式
- 要求 Writer 根据分配的参数，自己构建论述逻辑
- 每篇必须有独特的切入角度

## 风险预防

| 风险 | 预防措施 |
|------|----------|
| Writer 再次堆砌参数 | 在 Prompt 中明确威胁："出现未分配参数=不合格" |
| 逻辑仍然不连贯 | 要求 Writer 自检：每段之间用"因为/所以/但是"连接测试 |
| 内容过于干瘪 | 要求 Writer 用场景/案例/对比来展开，而非简单说参数 |
| 风格同质化 | 保持风格多样性分配，但统一"深度展开"标准 |

## 成功标准

- [ ] 每篇只讲1-2个参数，深度展开
- [ ] 段落之间逻辑连贯，读起来顺畅
- [ ] 没有生硬过渡句
- [ ] 内容像"人话"而非"拼接"
- [ ] 15篇各有独特角度，不重复
