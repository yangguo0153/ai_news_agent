# 10 条铁律

> Writer Agent 必须遵守。每条附验证方法，违反任何 1 条 = 不合格。

---

1. **数据不编造** — 正文中的所有数字、参数、价格只能来自"产品数据"段落。验证：搜索正文中所有数字，逐一比对产品数据，不存在产品数据中未提及的数字 = 通过。

2. **零第一人称** — 验证：搜索"我""我们""我的""咱们" = 0 个。

3. **零结构化** — 同一段落内"方面""这块"出现次数 ≤ 1。验证：搜索"方面""这块"，同段超过 1 个 = 不通过。

4. **零序号词** — 验证：搜索"首先""其次""最后""第三" = 0 个。

5. **零广告腔** — 验证：搜索"极致""震撼""卓越""匠心""硬核""拿捏" = 0 个。

6. **零车主伪装** — 验证：搜索"提车""首保""跑了趟长途""开了X年""开了X万公里" = 0 个。

7. **零虚构对话** — 验证：搜索"朋友说""销售说""车主说""有个车主" = 0 个。

8. **零 AI 句式** — 验证：搜索"但问题来了""算笔账就明白了""说白了""市场已经给出了答案""恰恰相反""这背后的逻辑值得聊聊""说实话""你看"（段落起始转折用法） = 0 个。

9. **短句控制** — 每句话 ≤ 25 个汉字（以句号/问号/感叹号断句计算）。超长句 ≤ 3 句/篇。

10. **字数达标** — 正文总字数在目标范围内（抖音 250-350 / 头条 650-800）。

11. **零能源类型** — 验证：搜索"混动""e:HEV""燃油""插混""纯电""油车""电车" = 0 个。可用"这款车""它""CR-V"指代。

12. **零时间限定** — 验证：搜索"刚买""新车""老车主""开了几年""开了一年""开两年""买的时候""当时买" = 0 个。不预设用户购车/用车时间点。

13. **首句点题** — 文章开头第一句必须直击核心卖点/主题，禁止绕弯子铺垫。验证：首句与 assigned topic 关联度是否明确。

14. **场景真实** — 验证：搜索"服务区看到""隔壁车位""旁边车的车主""路过看到""观察了一下" = 0 个。禁止虚构过度具体的第三方观察场景。
