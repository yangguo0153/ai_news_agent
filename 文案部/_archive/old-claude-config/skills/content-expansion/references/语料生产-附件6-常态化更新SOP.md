# 附件6：常态化更新SOP

> 语料库不是一次性工程，需要持续更新以保持内容新鲜度

---

## 一、更新频率总览

| 更新周期 | 语料类型 | 更新量 | 负责方 |
| -------- | ------------------------------ | ------------- | --------------------------- |
| 每周 | 热梗库、神评库 | 增3-5条/库 | OpenClaw |
| 每两周 | 开头/过渡/结尾变体库 | 每库增3-5条 | OpenClaw |
| 每月 | 风格样本 | 每风格增1-2篇 | OpenClaw |
| 每月 | 车型原子语料（种子库新增数据） | 按车型更新 | 人工提供数据 + OpenClaw整理 |
| 按需 | AI句式黑名单 | 发现即添加 | 人工审查 + OpenClaw执行 |

---

## 二、每周更新：热梗库 & 神评库

### 任务描述
从当周汽车领域热点中提取可复用的梗和神评句式。

### 来源
- 今日头条汽车频道热门评论
- 抖音汽车类热门视频评论区
- 懂车帝社区热帖
- 微博汽车话题

### 产出标准
- **热梗**：当周网络热梗中与汽车/消费/生活相关的，提取梗+使用场景+示例句
- **神评**：汽车评论区中有洞察力/有趣的评论，提取评论+适用话题

### 输出格式
```markdown
## 第X周热梗更新

### 热梗
| 梗 | 来源 | 使用场景 | 示例句 |
| ---- | ------ | -------- | ------ |
| [梗] | [来源] | [场景] | [示例] |

### 神评
| 原评论 | 适用话题 | 改写参考 |
| ------ | -------- | -------- |
| [评论] | [话题] | [改写] |
```

### 验收
- [ ] 热梗3-5条，每条含使用场景和示例句
- [ ] 神评3-5条，每条含适用话题和改写参考
- [ ] 零广告腔
- [ ] 口语化

## 三、每两周更新：开头/过渡/结尾变体库

### 任务描述
为三个变体库各新增3-5条变体，保持库的新鲜度。

### 产出标准
- 新增变体必须与库中现有变体**有差异**
- 可以从近期热点、季节变化、新话题中获取灵感
- 遵守附件1/2/3中的全部写作标准

### 执行步骤
1. 阅读当前变体库，了解已有变体
2. 确认哪些类型的变体较少（使用频率数据如果有的话）
3. 为使用较少或缺少变体的类型优先补充
4. 自检：新变体与现有变体无重复

### 验收
- [ ] 每库新增3-5条
- [ ] 与现有变体无重复
- [ ] 符合对应附件的写作标准

---

## 四、每月更新：风格样本

### 任务描述
为每种风格新增1-2篇完整范文，持续扩大风格理解面。

### 产出标准
- 遵守附件4中的全部标准
- 新增样本与现有全部样本在开头/结尾/结构上有差异
- 可以结合当月热点话题

### 执行步骤
1. 阅读当前各风格样本库
2. 统计各风格的开头类型、结尾类型、论证结构的使用分布
3. 优先选择**未被使用过的组合**
4. 写作并自检

### 验收
- [ ] 每风格新增1-2篇
- [ ] 开头/结尾/结构与现有样本不重复
- [ ] 650-800字（头条）
- [ ] 符合统一写作标准

## 五、每月更新：车型原子语料

### 任务描述
当有新的车型数据（月度销量更新、改款信息、新政策等）时，更新种子库。

### 执行步骤
1. 人工提供新数据（销量数据、改款信息、价格变动等）
2. OpenClaw将新数据整理到对应车型的种子库文件中
3. 基于新数据生成可直接使用的"数据+场景"原子语料

### 输出格式
```markdown
## 2026年X月更新

### 新增数据点
| 数据点 | 数值 | 来源 | 可用场景 |
| ------ | ---- | ------ | -------- |
| [数据] | [值] | [来源] | [场景] |

### 新增原子语料
1. [基于新数据的可用段落]
2. [基于新数据的可用段落]
...
```

## 六、按需更新：AI句式黑名单

### 触发条件
- Reviewer Agent 发现新的高频重复句式
- 人工审查发现批量产出中的"AI指纹"
- 用户/客户反馈某些句式过于模板化

### 执行步骤
1. 记录发现的新AI句式
2. 在以下文件中同步更新：
   - `references/10-rules.md` 第8条
   - `references/quality.md` 禁用高频AI句式列表 + 审校清单
   - `references/reviewer-prompt.md` 第8条检查项
3. 确认三处更新一致

### 验收
- [ ] 三个文件同步更新
- [ ] 新句式描述清晰，含使用场景说明（如"段落起始转折用法"）
