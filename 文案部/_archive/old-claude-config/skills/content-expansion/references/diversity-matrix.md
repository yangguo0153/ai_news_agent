# 多样化策略矩阵（7维锁定版）

## 目的

解决批量生产（100+篇/天）的内容雷同问题。通过7个维度的预锁定，确保任意连续5篇都不会出现完全相同的组合。

---

## 七维组合模型

### 维度1: 风格 (5种)

| ID  | 风格     | 特征                         | 起手式示例                                    |
| --- | -------- | ---------------------------- | --------------------------------------------- |
| S1  | 犀利观点 | 制造冲突、反问、揭露         | "就问一句..." / "别被忽悠了..." / "真相是..." |
| S2  | 干货科普 | 数据堆砌、专业分析、拆解     | "拆开底盘给你看..." / "三个数据说明问题..."   |
| S3  | 幽默段子 | 反转、自嘲、抖包袱           | "别人问买啥车..." / "丈母娘看了都说好..."     |
| S4  | 生活观察 | 场景代入、共情、细节描写     | "停车场看到个有意思的现象..."                 |
| S5  | 真实测评 | 第三视角客观记录、体验流水账 | "提车第一周，发现个事儿..."                   |

### 维度2: 角度 (6种)

| ID  | 角度     | 切入点                      | 适用场景                 |
| --- | -------- | --------------------------- | ------------------------ |
| A1  | 路人观察 | 旁观者视角，看别人的车/选择 | 停车场、4S店、小区       |
| A2  | 行业观察 | 从业者/分析师视角解读趋势   | 市场数据、行业动态、趋势 |
| A3  | 专业对比 | 技术拆解、参数对比          | 配置、底盘、动力         |
| A4  | 场景实测 | 具体场景验证                | 装载、油耗、空间         |
| A5  | 数据说话 | 油耗/价格/配置硬数据        | 经济性、性价比           |
| A6  | 成本算账 | 购车成本、用车成本分析      | 预算敏感型用户           |

### 维度3: 话题 (由 Brief 决定)

常见话题池：

- 春节返乡、日常通勤、家庭出行
- 年轻人首台车、家庭换购
- 对比竞品X、价格优势、配置优势
- 颜值、空间、动力、油耗

### 维度4: 开头类型 (5种)

| ID  | 类型       | 核心特征                 |
| --- | ---------- | ------------------------ |
| H1  | 反问型     | 以问题开头，挑战读者认知 |
| H2  | 揭秘型     | 以行业内幕/真相开头      |
| H3  | 数据冲击型 | 以数据开头，制造认知差   |
| H4  | 场景切入型 | 以具体场景描写开头       |
| H5  | 反常识型   | 以常识的反面开头         |

> 具体写法参考：`02-参考学习/风格语料库/开头变体库.md`（含每类型8个具体变体）

### 维度5: 论证结构 (6种)

| ID  | 结构              | 展开方式                     |
| --- | ----------------- | ---------------------------- |
| L1  | 数据先行→场景验证 | 先摆数据，再用场景证实       |
| L2  | 场景引入→数据佐证 | 先描述场景，再用数据支撑     |
| L3  | 对比法→结论       | 两方对比，自然得出结论       |
| L4  | 时间线→感受       | 按时间顺序展开，融入主观感受 |
| L5  | 问答递进→观点     | 层层设问推进，最终亮出观点   |
| L6  | 单一深挖→收尾     | 聚焦一个点深入展开，干脆收尾 |

### 维度6: 核心数据点 (从种子库提取)

每篇锁定1-2个核心数据点，同批次内不重复使用相同数据组合。

常见数据点池（根据种子库动态提取）：

- 保值率、月销量、油耗、百公里加速
- 后备箱容积、轴距、离地间隙
- 价格区间、优惠幅度、贷款月供
- 安全评级、质保政策、保养费用

### 维度7: 收尾方式 (5种)

| ID  | 方式     | 特征                     |
| --- | -------- | ------------------------ |
| E1  | 开放提问 | 抛出问题让读者思考/讨论  |
| E2  | 建议型   | 给出明确购买/选择建议    |
| E3  | 金句型   | 用一句有力的话收尾       |
| E4  | 数据回扣 | 回到开头提到的数据做总结 |
| E5  | 场景呼应 | 回到开头场景，形成闭环   |

> 具体写法参考：`02-参考学习/风格语料库/结尾变体库.md`（含每类型6个具体变体）

## 组合算法

### 总量计算

```
可用组合 = 5风格 × 6角度 × N话题 × 5开头 × 6结构 × M数据 × 5收尾
实际组合空间极大，足以支撑任意批量任务
```

### 分配规则

1. **预分配**：任务开始前，一次性生成所有篇的7维分配表，按分配表顺序创作
2. **均匀分布**：每种风格/角度/开头类型/收尾方式的使用次数尽量均衡
3. **连续5篇不重复**：任意连续5篇，7个维度都不允许出现完全相同的组合
4. **数据点不重复**：同批次内，核心数据点预分配不重复
5. **话题轮换**：批次间切换主话题
6. **动态调整**：每批次结束检查重复率，发现重复趋势时调整后续分配

### 分配表示例

```
篇1: S1+A1+T春节 | 开头:H4场景切入 | 结构:L2场景引入→数据佐证 | 数据:保值率 | 收尾:E1开放提问
篇2: S3+A6+T经济 | 开头:H1反问 | 结构:L5问答递进→观点 | 数据:油耗 | 收尾:E2建议型
篇3: S4+A4+T空间 | 开头:H5反常识 | 结构:L4时间线→感受 | 数据:后备箱容积 | 收尾:E5场景呼应
篇4: S2+A5+T动力 | 开头:H3数据冲击 | 结构:L1数据先行→场景验证 | 数据:百公里加速 | 收尾:E3金句型
篇5: S5+A3+T对比 | 开头:H2揭秘 | 结构:L3对比法→结论 | 数据:安全评级 | 收尾:E4数据回扣
篇6: S1+A2+T换购 | 开头:H3数据冲击 | 结构:L6单一深挖→收尾 | 数据:月销量 | 收尾:E1开放提问
篇7: S3+A5+T春节 | 开头:H4场景切入 | 结构:L3对比法→结论 | 数据:价格区间 | 收尾:E3金句型
...
```

## 防雷同检查

### 批次内检查

- [ ] 任意连续5篇内，无完全相同的7维组合
- [ ] 标题结构重复率 < 20%
- [ ] 开头前20字不重复
- [ ] 核心数据点无重复使用

### 跨批次检查

- [ ] 相邻批次话题不同
- [ ] 累计使用的风格分布均衡（无某风格占比>30%）
- [ ] 累计使用的角度分布均衡
- [ ] 累计使用的开头类型分布均衡
- [ ] 累计使用的收尾方式分布均衡

> 风格详解见对应语料库文件：`02-参考学习/风格语料库/抖音专属/` 和 `今日头条专属/`
