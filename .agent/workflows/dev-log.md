# 开发日志更新流程

每次开发任务完成后，**必须**执行此流程更新开发日志。

## 触发条件

- 完成Bug修复
- 完成功能开发或优化
- 发现重要的技术问题或解决方案

## 更新步骤

### 1. 定位项目日志文件

每个独立项目都应有以下两个文件：

- `.DEV_LOG.md` - 按日期记录的开发日志（类似changelog）
- `PROJECT_DEV_EXP.md` - 经验总结文档（按主题组织的最佳实践）

### 2. 更新 .DEV_LOG.md

在文件末尾添加新条目，格式：

```markdown
## YYYY-MM-DD

### 🐛 Bug 修复 / ✨ 功能增强 / 🔧 代码重构

- **问题描述**：[简述问题现象]
- **根因分析**：[分析为什么会出现这个问题]
- **解决方案**：[具体修复措施]
- **验证结果**：[如何确认修复成功]
```

### 3. 更新 PROJECT_DEV_EXP.md（可选）

如果发现了可复用的经验教训，添加到对应章节：

- 架构与流程优化
- Bug修复根因分析
- 最佳实践

### 4. 更新根目录索引

如果是新项目，在 `/Users/will/Desktop/通往AGI之路/README.md` 的开发日志索引表格中添加一行。

## ⚠️ 核心原则

> **不记录经验的开发毫无意义！**
>
> 每次踩过的坑、解决过的问题，都应该成为团队的知识资产。
