# 一、开场：AI能做什么了（5分钟）

## 1.1 AI能力跃迁

- 2024年：聊天助手（问答）
- 2025年：执行任务的Agent（能动手干活）
- 2026年：Skill时代——万物可AI化

## 1.2 核心前提

> **信息化是AI化的前提**

工作流没有数字化，AI就没有输入。
我们部门的工作是：帮各部门先完成**信息化**，再完成**AI化**。

## 1.3 现场示范

- "这份PPT是AI生成的"
- 说明用时对比：AI用了X小时 vs 传统方式X天

## 1.4 工具科普（简述）

- 国内模型：豆包、Deepseek、通义千问——免费、中文好、但深度分析弱
- 海外模型：Claude、Gemini、GPT——智能程度更高、复杂任务更强
- 关键认知：**模型只是工具，提示词工程才是核心竞争力**

# 二、现状诊断：公司AI应用水位（3分钟）

| 部门     | 现状                            | 2026年定位         |
| -------- | ------------------------------- | ------------------ |
| 策略策划 | 已用AI辅助创意，但工具层级低    | 教更好的工具和方法 |

# 三、2026年工作计划

---

## 3.1 文案部门赋能

### 现状痛点

1. **同质化严重**：3000条/月脚本高度重复，平台删稿率上升
2. **质量问题**：本田客户对AIGC脚本低质、广告味重多次反馈不满
3. **产能瓶颈**：3人月产2100-2400条，质量与数量难兼顾

### 根因分析

> 垃圾进，垃圾出

- 喂给AI的是PR味十足的官方brief
- 缺乏真实用户语料、网感素材、爆款结构参考
- 提示词模板单一，导致输出同质化

### 解决方案：搭建"文案大脑"系统

| 组件         | 作用                                                   |
| ------------ | ------------------------------------------------------ |
| 语料库       | 爆款脚本、真实用户评价、热梗、神评论——让AI学习真实网感 |
| 模板库       | 口播/混剪/资讯分类模板——标准化生产流程                 |
| 黑名单       | 广告味词汇自动过滤——降低PR感                           |
| 防同质化机制 | 角度轮换、语料轮换、开头策略变体——确保内容多样性       |

### 协作模式

```
AI创新部（大脑）          文案部（执行）
      │                        │
  生产提示词包 ──────→ 复制粘贴到AI工具
      │                        │
  维护语料库/模板 ←────── 反馈效果和问题
```

### 预期效果

| 指标       | 现状             | 目标          |
| ---------- | ---------------- | ------------- |
| 人均月产量 | 700-800条        | 900条（+15%） |
| 客户满意度 | 本田客户多次投诉 | 投诉清零      |
| 平台删稿率 | 较高             | 下降50%       |

## 3.2 舆情监测部门赋能

### 现状痛点

1. **人力瓶颈**：数据清洗人手不够，后续还需招聘
2. **标准缺失**：报告格式未统一，质量参差
3. **耗时长**：单份报告30-60分钟，临时需求响应压力大

### 解决方案

| 方向           | 内容                                        | 负责           |
| -------------- | ------------------------------------------- | -------------- |
| 数据清洗自动化 | 数据工程师写规则脚本，AI辅助识别噪音数据    | 数据工程师     |
| 报告模板标准化 | 统一PPT结构和文字风格，形成公司标准         | 与舆情部门共建 |
| 一键报告工具   | 内部工具——输入时间段+品牌，自动生成报告初稿 | 前端开发       |

### 协作模式

```
舆情部门                  AI创新部
    │                        │
 定义报告标准 ──────→ 开发生成工具
    │                        │
 人工审核润色 ←────── 工具输出初稿
```

### 预期效果

| 指标         | 现状      | 目标                               |
| ------------ | --------- | ---------------------------------- |
| 单份报告耗时 | 30-60分钟 | 15-30分钟（节省50%）               |
| 人力释放     | -         | 清洗人力需求减少，可不再扩招或少招 |
| 临时需求响应 | 压力大    | 快速响应                           |

## 3.3 媒介部门赋能

### 现状痛点

1. **工作量巨大**：月发布5000-8000次，账号筛选纯人工
2. **数据混乱**：多张表格散落，无统一账号库，无法追溯历史合作
3. **效率低下**：筛选100个账号约需1天，重复劳动多

### 前提判断

> **必须先信息化，才能AI化**

当前媒介数据涉及不同品牌客户需求、不同供应商能力，极度分散。
没有统一数据底座，AI无从下手。

### 解决方案：分两阶段走

**Phase 1：信息化——资产化、数据化**

| 任务         | 内容                                                       |
| ------------ | ---------------------------------------------------------- |
| 账号库建设   | 统一字段：平台、粉丝量、报价、历史合作记录、效果评估、标签 |
| 供应商库建设 | 统一字段：合作范围、价格区间、响应速度、历史评价           |
| 数据录入     | 媒介团队配合，逐步迁移现有数据                             |

**Phase 2：AI化——智能筛选**

| 任务         | 内容                                                       |
| ------------ | ---------------------------------------------------------- |
| 筛选工具开发 | 输入需求（品类、预算、平台、粉丝量区间）→ 系统推荐匹配账号 |
| 智能评估     | 基于历史合作数据，自动标记优质/风险账号                    |

### 协作模式

```
媒介部门                  AI创新部
    │                        │
 提供账号/供应商数据 ───→ 数据工程师建库
    │                        │
 使用筛选工具 ←─────────── 前端开发工具
    │                        │
 反馈使用问题 ───────────→ 持续优化
```

### 预期效果

| 指标       | 目标                                         |
| ---------- | -------------------------------------------- |
| 资产可视化 | 公司媒介资源、供应商资源全部数据化，一目了然 |
| 筛选效率   | 大幅提升（具体倍数待Phase 1完成后评估）      |
| 决策质量   | 历史数据可追溯，避免重复踩坑                 |

## 3.4 全员AI培训

### 分层设计

| 层级   | 对象      | 内容                                                 |
| ------ | --------- | ---------------------------------------------------- |
| 基础层 | 全员      | AI工具使用（豆包/Deepseek）、提示词基础技巧          |
| 进阶层 | 策划/文案 | 高级提示词工程、海外模型（Google AI Studio、Claude） |
| 专项层 | 各部门    | 定制培训（舆情用AI分析、媒介用AI筛选）               |

### 培训节奏

- **频率**：每季度1次集中培训
- **日常**：建立AI答疑群，随时响应
- **形式**：线下工作坊（动手实操）为主，效果远优于线上分享

### 目标

| 指标         | 现状                | 目标                     |
| ------------ | ------------------- | ------------------------ |
| 全员AI使用率 | 约30%               | 80%                      |
| 工具认知     | 只知道豆包/Deepseek | 了解并能使用海外高级模型 |
