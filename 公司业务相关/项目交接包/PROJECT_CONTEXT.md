# 三、2026年工作计划

## 部门使命

帮公司从"人力密集型"向"AI增强型"转型

## 三条主线（按优先级）

### 主线1：文案部门赋能

**现状痛点**：

- 3个文案，月产3000条脚本+1000篇资讯
- 同质化严重，平台删稿率高
- 本田客户多次投诉AIGC内容低质、广告味重
- 根因：喂给AI的是PR味brief，缺乏真实语料

**解决方案**：搭建"文案大脑"系统

- 语料库：爆款脚本、用户评价、热梗、神评论
- 模板库：口播/混剪/资讯分类模板
- 黑名单：广告味词汇自动过滤
- 防同质化机制：角度轮换、语料轮换

**协作模式**：

- AI创新部（大脑）生产提示词包
- 文案部（执行）复制粘贴到AI工具生成内容

**预期效果**：

- 产能：700-800条/人/月 → 900条（+15%）
- 质量：本田客户投诉清零
- 平台删稿率下降50%

### 主线2：舆情监测部门赋能

**现状痛点**：

- 数据清洗人力不够
- 报告格式未统一，单份报告30-60分钟
- 临时需求响应压力大

**解决方案**：

- 数据清洗自动化：规则脚本+AI辅助
- 报告模板标准化：统一PPT结构
- 一键报告工具：输入时间段+品牌，自动生成初稿

**预期效果**：

- 报告耗时：30-60分钟 → 15-30分钟（-50%）
- 释放人力，可减少招聘需求

### 主线3：媒介部门赋能

**现状痛点**：

- 月发布5000-8000次，账号筛选纯人工
- 数据混乱，多张表格散落，无统一账号库
- 筛选100个账号约需1天

**解决方案**：分两阶段

- Phase 1 信息化：统一账号库+供应商库（字段：平台、粉丝量、报价、历史合作、效果评估）
- Phase 2 AI化：智能筛选工具

**前提判断**：必须先信息化才能AI化，当前数据太乱

### 基础工作：全员AI培训

**分层设计**：

- 基础层（全员）：AI工具使用、提示词基础
- 进阶层（策划/文案）：高级提示词、海外模型
- 专项层（各部门）：定制培训

**节奏**：每季度1次集中培训 + 日常答疑群

**目标**：全员AI使用率从30%提升到80%

### 基础建设

| 方向           | 内容                                     |
| -------------- | ---------------------------------------- |
| 内部AI工具开发 | 文案大脑、舆情报告生成器、账号筛选工具   |
| 公司知识库建设 | 方法论、SOP、爆款案例数字化              |
| 数据资产沉淀   | 舆情历史数据、账号合作数据、内容效果数据 |

## 落地节奏

| 季度 | 重点工作                                |
| ---- | --------------------------------------- |
| Q1   | 文案大脑上线 + 全员基础培训             |
| Q2   | 舆情报告工具上线 + 媒介账号库信息化启动 |
| Q3   | 媒介AI筛选工具上线 + 进阶培训           |
| Q4   | 复盘优化 + 知识库建设                   |

# 四、文案大脑系统设计

## 核心架构

```
┌─────────────────────────────────────────────────────────┐
│                    大脑（AI创新部+Claude Code）           │
│  语料库 + 模板库 + 黑名单 → 生成"可执行提示词包"           │
└─────────────────────────┬───────────────────────────────┘
                          │
                          ↓ 输出提示词包（复制即用）
┌─────────────────────────────────────────────────────────┐
│                    执行（文案团队）                        │
│  提示词包 → 粘贴到豆包/Deepseek → 批量生成 → 人工微调      │
└─────────────────────────────────────────────────────────┘
```

## 文件结构

```
/文案大脑/
├── CLAUDE.md              # 大脑系统提示词
├── 黑名单.md              # 广告味词汇（已预填50+词）
├── 模板/
│   ├── 抖音口播.md
│   ├── 抖音混剪.md
│   └── 头条资讯.md
├── 语料库/
│   ├── 热梗.md
│   ├── 爆款-口播.md
│   ├── 爆款-混剪.md
│   ├── 神评论.md
│   └── 用户评价/
│       └── [车型].md
└── 输出/
```

## 文案部门现状

**工作流**：
接收业务方需求 → 消化ref → 喂给AI → 下提示词 → AI生成后人工修改 → 给到剪辑

**痛点**：

- 业务方brief太PR、广告味重
- 月产3000条脚本+1000篇资讯，同质化不可避免
- 被平台删除
- 缺乏网感、创意形式、真实用户语料

**现有提示词问题**：

- 太依赖"角色扮演"（让AI扮演网红）
- 没有给AI真实语料学习
- 模板单一导致输出同质化
